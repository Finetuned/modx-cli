Description:
  Install MODX 3.x into a target directory using composer or custom installers

Usage:
  install [options] [--] [<target> [<config>]]

Arguments:
  target                     Path to install MODX into (defaults to current directory)
  config                     Path to installer configuration file (for custom installers or setup)

Options:
      --installer=INSTALLER  Installer to use: "composer" (default) or custom name [default: "composer"]
      --installer-command=INSTALLER-COMMAND
                             Custom installer command line (use {target} and {config} placeholders)
      --modx-version=MODX-VERSION
                             MODX version for composer create-project (e.g., "3.0.5" or "latest")
      --composer-bin=COMPOSER-BIN
                             Composer executable or command [default: "composer"]
      --composer-no-interaction
                             Disable interactive prompts for composer
      --setup                Run MODX setup after installation
      --setup-command=SETUP-COMMAND
                             Custom setup command line (use {target} and {config} placeholders)
      --setup-config=SETUP-CONFIG
                             Path to setup config file (defaults to install config argument)
      --json                 Output results in JSON format
      --ssh=SSH              Run command on a remote server via SSH: [<user>@]<host>[:<port>][<path>]
  -h, --help                 Display help for the given command. When no command is given display help for the list command
  -q, --quiet                Do not output any message
  -V, --version              Display this application version
      --ansi|--no-ansi       Force (or disable --no-ansi) ANSI output
  -n, --no-interaction       Do not ask any interactive question
  -s, --site[=SITE]          An instance name to execute the command to
      --log-level=LOG-LEVEL  Set log level (debug, info, notice, warning, error, critical, alert, emergency)
      --log-file=LOG-FILE    Write logs to specified file
  -v|vv|vvv, --verbose       Increase the verbosity of messages: 1 for normal output, 2 for more verbose output and 3 for debug

Examples:
  # Install latest MODX 3.x in current directory
  modx install

  # Install specific MODX version
  modx install /var/www/mysite --modx-version=3.0.5

  # Install and run automated setup
  modx install /var/www/mysite --setup --setup-config=install-config.xml

  # Use Gitify as custom installer
  modx install /var/www/mysite config.xml --installer=gitify --installer-command="gitify modx:install {target} --config={config}"

For detailed documentation, see: docs/install.md
